# NexDynIPC 开环控制实现方案

## 1. 概述

本文档详细描述了在 NexDynIPC 项目中实现开环控制系统的完整方案。该方案参考了 Rigid-IPC 和 PolyFEM 的开环控制机制，与现有架构无缝集成。

### 1.1 开环控制定义

**开环控制（Open-Loop Control）** 是指控制指令完全由预设的轨迹或函数决定，不依赖于系统的实时状态反馈。在物理仿真中，这表现为：

- **运动学刚体（Kinematic Bodies）**：位置和朝向由预设轨迹决定，不受物理力影响
- **预设轨迹**：可以是关键帧动画、数学函数或外部数据驱动
- **硬约束执行**：通过约束求解器强制执行目标位姿

### 1.2 设计目标

1. **模块化设计**：轨迹、控制器、约束形式分离
2. **灵活扩展**：支持多种轨迹类型（固定、线性、关键帧、函数式）
3. **无缝集成**：与现有 IPCSolver 和 World 架构兼容
4. **高效求解**：使用增广拉格朗日法处理约束

### 3.1 Trajectory 轨迹系统

#### 3.1.1 基类定义

```cpp
// include/NexDynIPC/Control/Trajectory.h
#pragma once

#include <Eigen/Core>
#include <Eigen/Geometry>
#include <memory>
#include <vector>

namespace NexDynIPC::Control {

/**
 * @brief 轨迹基类，定义随时间变化的目标位姿
 * 
 * 开环控制的核心：预设轨迹，不依赖系统状态反馈
 */
class Trajectory {
public:
    virtual ~Trajectory() = default;

    /**
     * @brief 获取 t 时刻的目标位姿
     * @param t 当前时间
     * @param position 输出目标位置
     * @param orientation 输出目标朝向（四元数）
     */
    virtual void getPose(double t, 
                         Eigen::Vector3d& position, 
                         Eigen::Quaterniond& orientation) const = 0;

    /**
     * @brief 获取 t 时刻的目标速度
     * @param t 当前时间
     * @param linear_vel 输出线速度
     * @param angular_vel 输出角速度
     */
    virtual void getVelocity(double t,
                             Eigen::Vector3d& linear_vel,
                             Eigen::Vector3d& angular_vel) const = 0;
};

} // namespace NexDynIPC::Control
```

### 3.2 KinematicBody 运动学刚体

#### 3.2.1 类定义

```cpp
// include/NexDynIPC/Control/KinematicBody.h
#pragma once

#include "NexDynIPC/Dynamics/RigidBody.h"
#include "NexDynIPC/Control/Trajectory.h"
#include <memory>

namespace NexDynIPC::Control {

/**
 * @brief 运动学刚体：受开环控制，不受物理力影响
 * 
 * 特性：
 * - 质量和惯性仅用于碰撞响应计算（影响其他动态物体）
 * - 位置和朝向由预设轨迹决定，不由牛顿-欧拉方程求解
 * - 通过约束形式强制执行目标位姿
 */
class KinematicBody : public Dynamics::RigidBody {
public:
    KinematicBody();
    
    /**
     * @brief 设置控制轨迹
     */
    void setTrajectory(std::shared_ptr<Trajectory> trajectory);
    
    /**
     * @brief 获取当前轨迹
     */
    std::shared_ptr<Trajectory> getTrajectory() const { return trajectory_; }
    
    /**
     * @brief 更新目标位姿（由 OpenLoopController 调用）
     * @param t 当前时间
     */
    void updateTargetPose(double t);
    
    /**
     * @brief 获取目标位置（用于约束计算）
     */
    const Eigen::Vector3d& getTargetPosition() const { return target_position_; }
    
    /**
     * @brief 获取目标朝向（用于约束计算）
     */
    const Eigen::Quaterniond& getTargetOrientation() const { return target_orientation_; }

    /**
     * @brief 获取目标速度（用于预测位置计算）
     */
    const Eigen::Vector3d& getTargetVelocity() const { return target_velocity_; }
    
    const Eigen::Vector3d& getTargetAngularVelocity() const { return target_angular_velocity_; }

    /**
     * @brief 应用目标位姿到实际状态（在求解后调用）
     */
    void applyTargetPose();

private:
    std::shared_ptr<Trajectory> trajectory_;
    
    // 目标状态（由轨迹计算得到）
    Eigen::Vector3d target_position_;
    Eigen::Quaterniond target_orientation_;
    Eigen::Vector3d target_velocity_;
    Eigen::Vector3d target_angular_velocity_;
};

} // namespace NexDynIPC::Control
```

#### 3.2.2 实现要点

```cpp
// src/Control/KinematicBody.cpp

void KinematicBody::updateTargetPose(double t) {
    if (!trajectory_) return;
    
    // 从轨迹获取目标位姿和速度
    trajectory_->getPose(t, target_position_, target_orientation_);
    trajectory_->getVelocity(t, target_velocity_, target_angular_velocity_);
    
    // 归一化四元数确保数值稳定
    target_orientation_.normalize();
}

void KinematicBody::applyTargetPose() {
    // 求解后，将目标位姿应用到实际状态
    // 这确保了运动学体严格遵循轨迹
    position = target_position_;
    orientation = target_orientation_;
    velocity = target_velocity_;
    angular_velocity = target_angular_velocity_;
}
```

### 3.3 KinematicConstraintForm 运动学约束形式

#### 3.3.1 理论基础

使用**增广拉格朗日法（Augmented Lagrangian Method）**强制执行运动学约束：

**约束定义**：
```
C(x) = [p - p_target; θ - θ_target] = 0
```

其中：
- `p`：当前位置（优化变量）
- `p_target`：目标位置（由轨迹决定）
- `θ`：当前旋转参数（旋转向量）
- `θ_target`：目标旋转参数

**能量函数**：
```
E = λᵀ·C + 0.5·μ·Cᵀ·C
```

其中：
- `λ`：拉格朗日乘子（6维向量）
- `μ`：惩罚刚度（标量，越大约束越严格）

**梯度**：
```
∇E = Jᵀ·(λ + μ·C)
```

其中 `J = ∂C/∂x` 是约束雅可比矩阵。

**Hessian（高斯-牛顿近似）**：
```
H ≈ Jᵀ·μ·J
```

**ALM 更新规则**：
```
λ ← λ + μ·C
```

#### 3.3.2 类定义

```cpp
// include/NexDynIPC/Control/KinematicConstraintForm.h
#pragma once

#include "NexDynIPC/Dynamics/Forms/Form.h"
#include "NexDynIPC/Control/KinematicBody.h"
#include <vector>
#include <memory>

namespace NexDynIPC::Control {

/**
 * @brief 运动学约束形式：使用增广拉格朗日法强制执行目标位姿
 * 
 * 能量函数：E = λ·C + 0.5·μ·Cᵀ·C
 * - C = [p - p_target; θ - θ_target] （6维约束）
 * - λ: 拉格朗日乘子
 * - μ: 惩罚刚度（增大以提高约束精度）
 */
class KinematicConstraintForm : public Dynamics::Form {
public:
    KinematicConstraintForm();
    ~KinematicConstraintForm() override = default;

    /**
     * @brief 添加运动学刚体到约束系统
     */
    void addKinematicBody(std::shared_ptr<KinematicBody> body);
    
    /**
     * @brief 移除运动学刚体
     */
    void removeKinematicBody(std::shared_ptr<KinematicBody> body);
    
    /**
     * @brief 更新所有运动学体的全局索引（在 World 构建后调用）
     */
    void updateGlobalIndices(const std::vector<std::shared_ptr<Dynamics::RigidBody>>& bodies);

    /**
     * @brief 更新拉格朗日乘子（ALM 迭代）
     */
    void updateLambdas(const Eigen::VectorXd& x);

    /**
     * @brief 重置所有乘子
     */
    void resetLambdas();

    // Form 接口实现
    double value(const Eigen::VectorXd& x) const override;
    void gradient(const Eigen::VectorXd& x, Eigen::VectorXd& grad) const override;
    void hessian(const Eigen::VectorXd& x, std::vector<Eigen::Triplet<double>>& triplets) const override;

    /**
     * @brief 设置惩罚刚度
     */
    void setStiffness(double mu) { mu_ = mu; }
    double getStiffness() const { return mu_; }

private:
    struct KinematicConstraint {
        std::shared_ptr<KinematicBody> body;
        int global_idx = -1;        // 在全局状态向量中的起始索引
        Eigen::VectorXd lambda;     // 拉格朗日乘子（6维）
    };

    std::vector<KinematicConstraint> constraints_;
    double mu_ = 1e8;  // 默认惩罚刚度
};

} // namespace NexDynIPC::Control
```

#### 3.3.3 实现要点

```cpp
// src/Control/KinematicConstraintForm.cpp

double KinematicConstraintForm::value(const Eigen::VectorXd& x) const {
    double energy = 0.0;
    
    for (const auto& constraint : constraints_) {
        if (constraint.global_idx < 0) continue;
        
        const auto& body = constraint.body;
        int idx = constraint.global_idx;
        
        // 计算位置约束：C_pos = p - p_target
        Eigen::Vector3d p = x.segment<3>(idx);
        Eigen::Vector3d C_pos = p - body->getTargetPosition();
        
        // 计算旋转约束：C_rot = θ - θ_target
        // 注意：这里假设 x 存储的是旋转向量增量
        Eigen::Vector3d theta = x.segment<3>(idx + 3);
        // θ_target 需要从目标朝向计算
        Eigen::Vector3d theta_target = computeThetaTarget(body->getTargetOrientation());
        Eigen::Vector3d C_rot = theta - theta_target;
        
        // 组装约束向量
        Eigen::VectorXd C(6);
        C << C_pos, C_rot;
        
        // 计算能量：λ·C + 0.5·μ·C·C
        energy += constraint.lambda.dot(C) + 0.5 * mu_ * C.squaredNorm();
    }
    
    return energy;
}

void KinematicConstraintForm::gradient(const Eigen::VectorXd& x, Eigen::VectorXd& grad) const {
    for (const auto& constraint : constraints_) {
        if (constraint.global_idx < 0) continue;
        
        const auto& body = constraint.body;
        int idx = constraint.global_idx;
        
        // 计算约束 C
        Eigen::VectorXd C(6);
        // ... 计算 C_pos 和 C_rot
        
        // 计算梯度：∇E = Jᵀ·(λ + μ·C)
        // 对于位置约束：J_pos = I₃
        // 对于旋转约束：J_rot = I₃
        Eigen::VectorXd force = constraint.lambda + mu_ * C;
        
        grad.segment<3>(idx) += force.segment<3>(0);      // 位置部分
        grad.segment<3>(idx + 3) += force.segment<3>(3);  // 旋转部分
    }
}

void KinematicConstraintForm::hessian(const Eigen::VectorXd& x, 
                                       std::vector<Eigen::Triplet<double>>& triplets) const {
    for (const auto& constraint : constraints_) {
        if (constraint.global_idx < 0) continue;
        
        int idx = constraint.global_idx;
        
        // Hessian：H = Jᵀ·μ·J = μ·I₆
        // 位置部分
        for (int i = 0; i < 3; ++i) {
            triplets.emplace_back(idx + i, idx + i, mu_);
        }
        // 旋转部分
        for (int i = 0; i < 3; ++i) {
            triplets.emplace_back(idx + 3 + i, idx + 3 + i, mu_);
        }
    }
}

void KinematicConstraintForm::updateLambdas(const Eigen::VectorXd& x) {
    for (auto& constraint : constraints_) {
        if (constraint.global_idx < 0) continue;
        
        const auto& body = constraint.body;

        // ... 计算 C
        
        // 更新乘子：λ ← λ + μ·C

    }
}
```

#### 3.4.1 类定义

```cpp
// include/NexDynIPC/Control/OpenLoopController.h
#pragma once
```

```cpp
        body->updateTargetPose(t);
```

#### 4.1.2 实现修改

```cpp
// src/Dynamics/IPCSolver.cpp

void IPCSolver::step(World& world, double dt) {
    static double current_time = 0.0;  // 或使用成员变量
    
    // ========== 1. 开环控制预求解更新 ==========
    if (ol_controller_) {
        ol_controller_->preSolveUpdate(current_time, dt);
    }
    
    // ========== 2. 构建优化问题 ==========
    // 收集所有能量形式
    std::vector<std::shared_ptr<Form>> all_forms = world.forms;
    
    // 添加运动学约束形式
    if (ol_controller_) {
        all_forms.push_back(ol_controller_->getConstraintForm());
    }
    
    // 构建总能量函数 E(x) = Σ E_i(x)
    auto total_energy = [&](const Eigen::VectorXd& x) -> double {
        double E = 0.0;
        for (auto& form : all_forms) {
            E += form->value(x);
        }
        return E;
    };
    
    // 构建梯度
    auto total_gradient = [&](const Eigen::VectorXd& x, Eigen::VectorXd& grad) {
        grad.setZero(x.size());
        for (auto& form : all_forms) {
            form->gradient(x, grad);
        }
    };
    
    // 构建 Hessian
    auto total_hessian = [&](const Eigen::VectorXd& x, 
                              std::vector<Eigen::Triplet<double>>& triplets) {
        for (auto& form : all_forms) {
            form->hessian(x, triplets);
        }
    };
    
    // ========== 3. ALM 迭代求解 ==========
    const int max_alm_iterations = 3;
    
    for (int alm_iter = 0; alm_iter < max_alm_iterations; ++alm_iter) {
        // 牛顿迭代求解当前 ALM 子问题
        Eigen::VectorXd x = newton_solver_.solve(total_energy, total_gradient, total_hessian);
        
        // 更新拉格朗日乘子
        if (ol_controller_) {
            ol_controller_->getConstraintForm()->updateLambdas(x);
        }
        
        // 检查约束收敛
        // ...
    }
    
    // ========== 4. 开环控制后求解更新 ==========
    if (ol_controller_) {
        ol_controller_->postSolveUpdate();
    }
    
    // ========== 5. 更新状态 ==========
    // 更新速度、加速度等
    // ...
    
    current_time += dt;
}
```

```cpp
std::shared_ptr<Control::Trajectory> SceneLoader::parseTrajectory(const nlohmann::json& traj_json) {
    std::string type = traj_json.value("type", "fixed");
    
    if (type == "fixed") {
        auto pos = traj_json["position"];
        return std::make_shared<Control::FixedTrajectory>(
            Eigen::Vector3d(pos[0], pos[1], pos[2])
        );
    } else if (type == "linear") {
        auto vel = traj_json["velocity"];
        auto start = traj_json.value("start", std::vector<double>{0, 0, 0});
        return std::make_shared<Control::LinearTrajectory>(
            Eigen::Vector3d(start[0], start[1], start[2]),
            Eigen::Vector3d(vel[0], vel[1], vel[2])
        );
    } else if (type == "keyframes") {
        auto traj = std::make_shared<Control::KeyframeTrajectory>();
        double fps = traj_json.value("fps", 30.0);
        
        for (const auto& frame : traj_json["frames"]) {
            Control::KeyframeTrajectory::Keyframe keyframe;
            keyframe.time = frame["time"];
            
            auto pos = frame["position"];
            keyframe.position = Eigen::Vector3d(pos[0], pos[1], pos[2]);
            
            if (frame.contains("orientation")) {
                auto ori = frame["orientation"];
                keyframe.orientation = Eigen::Quaterniond(ori[3], ori[0], ori[1], ori[2]);
            }
            
            traj->addKeyframe(keyframe);
        }
        
        return traj;
    }
    
    // 默认返回固定轨迹
    return std::make_shared<Control::FixedTrajectory>(Eigen::Vector3d::Zero());
}
```

```cpp

    config.scene_file = "scene_with_kinematics.json";
    
    // 2. 创建世界、求解器和开环控制器
    Dynamics::World world;
    auto solver = std::make_shared<Dynamics::IPCSolver>();

    ol_controller->updateGlobalIndices(world.bodies);
    
    // 6. 运行仿真
    double t = 0.0;
    while (t < config.max_time) {
        solver->step(world, config.dt);
        t += config.dt;
    }
    
    return 0;
}
```

## 7. 关键技术要点

### 7.1 增广拉格朗日法参数选择

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| μ（惩罚刚度） | 1e8 ~ 1e12 | 越大约束越严格，但可能导致数值问题 |
| ALM 迭代次数 | 3 ~ 5 | 平衡精度和效率 |
| 乘子重置策略 | 每时间步重置 | 可加速收敛，但可能降低精度 |

### 7.2 数值稳定性

1. **四元数归一化**：每次更新后归一化目标朝向
2. **旋转向量限制**：确保旋转增量在合理范围内（< π）
3. **Hessian 正则化**：添加小量对角项防止奇异

### 7.3 性能优化

1. **约束稀疏性**：每个运动学体只影响 6 个 DOF
2. **预计算**：关键帧轨迹可预计算插值系数
3. **并行更新**：多个运动学体的目标位姿更新可并行

### 8.1 支持更多轨迹类型

- **贝塞尔曲线**：平滑轨迹

### 8.3 闭环控制扩展

- **PID 控制器**：基于误差反馈调整
- **模型预测控制（MPC）**：考虑未来轨迹优化

## 9. 总结

本方案提供了一个完整的开环控制系统实现，具有以下特点：

1. **模块化**：轨迹、控制器、约束分离，易于扩展
2. **兼容性**：与现有 IPCSolver 和 World 架构无缝集成
3. **灵活性**：支持多种轨迹定义方式
4. **鲁棒性**：使用增广拉格朗日法确保约束精度

通过本方案，NexDynIPC 可以支持复杂的运动学控制场景，如机械臂操作、动画角色、移动平台等。
