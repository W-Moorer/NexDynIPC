# 旋转副匀速转动实现方案

## 问题描述

给定一个旋转副（RevoluteJoint），如何实现其以**指定速度匀速转动**？

## 物理分析

### 旋转副动力学

旋转副连接两个刚体（父刚体 body_a 和子刚体 body_b），其运动方程为：

```
I * α = τ_motor - τ_friction - τ_gravity - τ_constraint
```

其中：
- `I`：等效转动惯量
- `α`：角加速度
- `τ_motor`：电机驱动力矩
- `τ_friction`：摩擦力矩
- `τ_gravity`：重力产生的力矩
- `τ_constraint`：约束力矩

### 匀速转动条件

要实现匀速转动（角加速度 α = 0），需要：

```
τ_motor = τ_friction + τ_gravity + τ_constraint
```

即电机力矩需要抵消所有外部力矩。

## 实现方案

### 方案一：速度伺服控制（推荐）

使用 PD 控制器根据速度误差计算力矩：

```
τ = kp * (ω_target - ω_current) + kd * (0 - α_current)
```

**优点**：
- 实现简单
- 可以补偿摩擦和负载变化
- 速度跟踪精度高

**缺点**：
- 需要调参（kp, kd）
- 存在稳态误差（可用积分项消除）

### 方案二：直接力矩补偿

计算维持匀速所需的力矩并直接施加：

```
τ = τ_friction + τ_gravity
```

**优点**：
- 理论上可以实现完美匀速
- 无振荡

**缺点**：
- 需要精确知道摩擦力和重力矩
- 对参数变化敏感

### 方案三：运动学约束（Kinematic Control）

将旋转副的一个刚体设为 KinematicBody，通过轨迹控制角度：

```
θ(t) = θ_0 + ω_target * t
```

**优点**：
- 精确控制位置
- 无动力学响应延迟

**缺点**：
- 不是力控制，无法模拟真实电机
- 对碰撞响应不真实

## 详细实现

### 方案一实现：VelocityServoController

```cpp
class VelocityServoController {
public:
    // 设置目标速度
    void setTargetVelocity(double omega_target);
    
    // 更新控制（每帧调用）
    double update(double dt) {
        // 获取当前状态
        double omega_current = joint_->getAngularVelocity();
        double theta_current = joint_->getAngle();
        
        // 计算速度误差
        double error = omega_target_ - omega_current;
        
        // PD控制
        double tau = kp_ * error - kd_ * omega_current;
        
        // 施加力矩限制
        tau = std::clamp(tau, tau_min_, tau_max_);
        
        // 施加到关节
        joint_->applyTorque(tau);
        
        return tau;
    }

private:
    RevoluteJoint* joint_;
    double omega_target_ = 0.0;
    double kp_ = 100.0;  // 位置增益
    double kd_ = 10.0;   // 速度增益
    double tau_min_ = -100.0;
    double tau_max_ = 100.0;
};
```

### 方案二实现：DirectTorqueController

```cpp
class DirectTorqueController {
public:
    void update(double dt) {
        // 计算重力矩
        double tau_gravity = computeGravityTorque();
        
        // 计算摩擦力矩
        double tau_friction = computeFrictionTorque();
        
        // 总补偿力矩
        double tau = tau_gravity + tau_friction;
        
        // 可选：添加小的速度修正项
        double omega = joint_->getAngularVelocity();
        tau += kp_ * (omega_target_ - omega);
        
        joint_->applyTorque(tau);
    }

private:
    double computeGravityTorque() {
        // τ = r × F_gravity
        // 需要知道质心位置和重力方向
        Vector3d r = child_body_->getCenterOfMass() - joint_position_;
        Vector3d F_gravity = Vector3d(0, -9.8 * child_body_->mass, 0);
        Vector3d tau = r.cross(F_gravity);
        return tau.dot(joint_axis_);
    }
    
    double computeFrictionTorque() {
        // 库仑摩擦 + 粘性摩擦
        double omega = joint_->getAngularVelocity();
        double tau_coulomb = (omega > 0) ? mu_coulomb_ : -mu_coulomb_;
        double tau_viscous = mu_viscous_ * omega;
        return tau_coulomb + tau_viscous;
    }
};
```

### 方案三实现：KinematicRotation

```cpp
class KinematicRotation {
public:
    void update(double t) {
        // 计算目标角度
        double theta = theta_0_ + omega_target_ * t;
        
        // 设置子刚体的朝向
        Quaterniond q = AngleAxisd(theta, joint_axis_);
        child_body_->setOrientation(q);
    }
};
```

## 参数调优建议

### PD控制器参数

对于旋转副速度控制，建议参数：

```cpp
// 小负载（如轻杆）
kp = 50.0;
kd = 5.0;

// 中等负载
kp = 100.0;
kd = 10.0;

// 大负载（如机械臂）
kp = 200.0;
kd = 20.0;
```

### 调优原则

1. **增大 kp**：响应更快，但可能振荡
2. **增大 kd**：抑制振荡，但响应变慢
3. **力矩限制**：保护电机和机械结构

## 完整使用示例

```cpp
// 创建旋转副
auto joint = std::make_shared<RevoluteJoint>(body_a, body_b, axis, position);
world->addJoint(joint);

// 方案一：速度伺服控制
auto servo = std::make_shared<VelocityServoController>(joint.get());
servo->setTargetVelocity(1.0);  // 1 rad/s
servo->setGains(100.0, 10.0);   // kp, kd

// 仿真循环
for (double t = 0; t < 10.0; t += dt) {
    servo->update(dt);
    solver->step(world, dt);
}
```

## 性能对比

| 方案 | 跟踪精度 | 实现复杂度 | 计算成本 | 适用场景 |
|------|---------|-----------|---------|---------|
| 速度伺服 | 高 | 低 | 低 | 大多数场景 |
| 直接力矩 | 极高 | 高 | 中 | 精密控制 |
| 运动学约束 | 完美 | 低 | 低 | 纯动画/无碰撞 |

## 推荐方案

对于大多数机械手夹取场景，**推荐方案一（速度伺服控制）**，原因：

1. 实现简单，不需要精确建模
2. 对摩擦和负载变化有鲁棒性
3. 可以通过调参平衡响应速度和稳定性
4. 易于扩展到多个关节
