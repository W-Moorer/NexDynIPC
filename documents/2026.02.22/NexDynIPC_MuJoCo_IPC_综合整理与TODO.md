# NexDynIPC vs MuJoCo/IPC：完整整理与 TODO（效率优先）

更新时间：2026-02-22

## 1. 文档目的与范围

本文将此前讨论的内容统一整理到一个可执行文档中，覆盖四个问题：
1. 当前项目（NexDynIPC）与 `temps/mujoco-main` 的推导与工程流程差异是什么。
2. 二者各自优劣是什么。
3. 从“计算效率”和“计算精度”两个维度，当前项目该如何优化。
4. 采用“MuJoCo 主流程 + IPC 局部精化”的混合路线应如何设计与落地，以及是否有相关研究支撑。

本文定位为“研发执行稿”：不仅给结论，也给分阶段 TODO 和验收指标。

## 2. 一句话结论

“图形学能量法”方向本身没有问题；但在大规模或高吞吐仿真场景下，MuJoCo 式约束求解主流程通常更高效。效率优先时，建议采用混合路线：
1. 全局走 MuJoCo 式快速主流程保证吞吐与稳定时延。
2. 局部保留 IPC 能量项对高风险接触进行有限预算精化，以补精度与鲁棒性。

## 3. 当前项目 vs MuJoCo：详细差异

### 3.1 数学建模与未知量

NexDynIPC（当前实现）核心是离散步上的总能量最小化：
`min_x E_total(x) = E_inertia + s*E_potential + E_constraints + s*E_extra`

MuJoCo 核心是动力学方程与约束力求解分离：
`M * vdot + c = tau + J^T * f`
其中约束力 `f` 通过约束优化/求解器体系得到。

差异本质：
1. NexDynIPC 直接对位姿增量（尤其是最大坐标下的刚体 DOF）做优化。
2. MuJoCo 以动力学主方程为核心，再在约束空间求解接触/摩擦/关节等约束力。

### 3.2 约束组织方式

NexDynIPC 当前主要采用 ALM（增广拉格朗日）能量项处理关节约束，典型形态：
`E = lambda^T C + 0.5 * mu * ||C||^2`

MuJoCo 将等式约束、接触、限位、摩擦损失等统一纳入约束求解框架，并提供多种求解策略（如 PGS/CG/Newton 路线）。

### 3.3 接触与摩擦实现成熟度

NexDynIPC 已有 barrier/CCD/摩擦项框架，但现阶段存在工程简化点：
1. 摩擦项中存在固定 `dt` 的简化写法风险（应使用真实时间步长）。
2. CCD 旋转推进存在待补全项（角速度引起的旋转更新精度不足）。
3. 主流程中接触法向力估计与精化触发尚未完全形成“预算驱动闭环”。

MuJoCo 在接触生成、筛选、求解器配合、参数化（如 `solref/solimp`）上更成熟，实战稳定性更强。

### 3.4 工程化与可扩展性侧重点

1. NexDynIPC 优势在于统一能量框架，便于引入新物理项和研究性扩展。
2. MuJoCo 优势在于工业化求解管线，warmstart/island/多求解器切换等效率特性更完整。

## 4. 二者优劣对比（直接结论）

### 4.1 NexDynIPC 优势

1. 统一能量表达，模型扩展路径清晰。
2. 对复杂接触能量项、几何细节项更友好。
3. 研究原型迭代快，适合做新方法验证。

### 4.2 NexDynIPC 短板

1. 全局大问题牛顿迭代在大规模场景成本高。
2. ALM 外循环 + 接触复杂性叠加时，迭代预算压力大。
3. 目前接触/摩擦细节与鲁棒工程闭环仍在建设中。

### 4.3 MuJoCo 优势

1. 约束求解主流程高效，吞吐和实时性表现通常更好。
2. 成熟的接触处理链路，工程可控性强。
3. 对机器人动力学场景适配深，稳定性经验丰富。

### 4.4 MuJoCo 短板

1. 以机器人动力学主线为核心，某些图形学细粒度能量项整合门槛较高。
2. 直接承载“高精度几何非穿透精化”时，不如 IPC 能量法直观。

## 5. 计算效率分析（效率优先）

### 5.1 结论

在“规模增大 + 接触密集”场景，MuJoCo 式主流程通常优于纯全局能量最小化。原因不是“理论更高级”，而是“工程路径更贴合实时预算”：
1. 约束空间求解更利于迭代预算控制。
2. warmstart + island 并行 + 早停策略可形成稳定时延。
3. 局部高风险问题单独精化，避免把全部复杂度推给全局问题。

### 5.2 当前项目的效率关键瓶颈（按优先级）

1. 全局 Hessian 组装与分解成本在复杂场景上升快。
2. ALM 外循环次数受约束残差与接触刚性影响明显。
3. 缺少“风险驱动局部求精”，导致本可局部处理的问题进入全局迭代。
4. 统计与预算反馈不足，难以形成稳定帧时间控制。

## 6. 计算精度分析（在效率约束下提升）

### 6.1 当前可利用的精度优势

1. 能量一致性框架使物理项叠加和梯度/Hessian 推导路径明确。
2. 对几何接触惩罚与势能项可表达能力强。

### 6.2 需要优先补齐的精度短板

1. 摩擦项时间步长一致性（固定 dt 问题必须修正）。
2. CCD 旋转推进完整性（角速度路径补全）。
3. 接触法向/切向耦合精化的触发条件与终止条件需要量化。

### 6.3 精度提升原则（效率不退化）

1. 不做“全局精度一刀切提升”，只对高风险簇精化。
2. 精化过程必须受预算约束，有超时降级策略。
3. 先修“系统性误差源”（dt、CCD），再加“昂贵高阶模型”。

## 7. 混合路线设计：MuJoCo 主流程 + IPC 局部精化

### 7.1 三层架构

1. Global Fast Layer：全局快速推进与约束求解。
2. Local Refinement Layer：仅对高风险接触簇做 IPC 局部精化。
3. Policy & Budget Layer：决定本步精化范围、迭代上限、降级策略。

### 7.2 单步流程（推荐）

1. Stage A：运动学与惯性更新。
2. Stage B：接触候选生成与约束分岛。
3. Stage C：全局快速求解（可 warmstart，支持 early-stop）。
4. Stage D：风险评分，选择 Top-K 接触簇。
5. Stage E：对选中簇执行 IPC 局部牛顿精化（有限迭代）。
6. Stage F：状态合并与质量检查（最小距离、残差、能量异常）。
7. Stage G：预算回路（超时即降级，富余则提升精化额度）。

### 7.3 风险评分建议

可使用：
`risk = w1*penetration_risk + w2*relative_normal_speed + w3*friction_slip_risk + w4*residual`

触发策略：
1. 仅 `risk > T_refine` 的簇进入 IPC。
2. 按分数排序，仅取 Top-K。
3. 多帧持续高风险簇提高优先级。

### 7.4 预算控制建议

每步预算拆分：
1. `B_total`：总预算。
2. `B_fast`：全局快速层预算。
3. `B_refine`：局部精化预算。
4. `B_guard`：日志与保护预算。

动态降级：
1. 全局超时：降低主迭代上限或候选接触上限。
2. 精化超时：立即停止后续簇，保留已完成修正。
3. 连续超时 N 帧：进入效率保护模式。

## 8. 相关研究全网检索结论（截至 2026-02-22）

### 8.1 总结判断

1. “MuJoCo 主流程 + IPC 局部精化”作为明确标准方案，公开资料中尚未看到行业统一范式。
2. 但“高效主流程 + 局部高精度接触修正”的思想在相关研究中广泛存在，属于可行且有研究基础的组合方向。
3. 该路线有机会形成你项目的差异化贡献点，尤其在“效率-精度预算闭环”与“工程可复现”两方面。

## 9. TODO 清单（按优先级落地）

### 9.1 P0（立即，1-2 周）

1. 修正摩擦项时间步长一致性，移除固定 `dt` 假设。
2. 补齐 CCD 旋转推进路径，消除角速度漏项。
3. 增加统一统计：每步时间、迭代次数、约束残差、最小距离。
4. 建立基础 warmstart 缓存（上一帧约束力/乘子或近似状态）。

验收标准：
1. 结果正确性不下降。
2. 单步耗时波动可观测，日志可追踪。

### 9.2 P1（短期，2-4 周）

1. 接触分岛与接触簇构建。
2. 风险评分 + Top-K 选择器。
3. `LocalIPCRefiner`（仅局部 DOF 子问题）原型。
4. 预算控制器与超时降级策略。

验收标准：
1. 在固定场景下，平均步时下降或持平。
2. `min_distance` 与约束残差优于“仅全局快解”。

### 9.3 P2（中期，4-8 周）

1. 引入多求解器策略切换（快解器 + 局部牛顿精化）。
2. 完善摩擦锥/参数化模型与稳定化策略。
3. 构建回归基准场景库（密集接触、堆叠、夹持、快速碰撞）。

验收标准：
1. 吞吐、稳定性、精度三项指标均可量化提升。
2. 混合路线在至少两类场景表现优于单一路线。

## 10. 评测与验收指标（必须量化）

### 10.1 效率指标

1. 平均步时（ms/step）。
2. P95/P99 步时（时延稳定性）。
3. 每步迭代次数（全局 + 局部）。
4. 每秒可模拟步数（SPS）。

### 10.2 精度指标

1. 全局最小间隙 `min_distance`。
2. 约束残差范数。
3. 穿透事件计数与最大穿透深度。
4. 能量漂移统计（长期稳定性）。

### 10.3 成功判据（建议）

1. 相比当前主干：平均步时降低 >= 20%。
2. 穿透事件减少 >= 50%。
3. 高风险场景下 `min_distance` 显著改善且不出现明显帧时爆炸。

## 11. 最终建议（效率优先下提高精度）

按以下顺序执行最稳妥：
1. 先修基础正确性：`dt` 一致性 + CCD 旋转推进。
2. 再做效率骨架：warmstart + 分岛 + 预算统计。
3. 最后上局部精化：风险驱动 Top-K IPC，仅在需要时触发。

这条路径可以在不推翻当前能量框架的前提下，吸收 MuJoCo 流程的高效率优势，并保留 IPC 在复杂接触下的精度兜底能力。
