# NexDynIPC 作为“完整动力学仿真系统”的缺口分析（2026-02-27）

## 0. 当日代码实装核对（与迁移规划对齐）

本节用于回答“`documents/2026.02.27/stark约束与关节工程化迁移规划.md` 当前完成到哪一步”。

### 0.1 迁移规划完成度（截至当前）

已完成（代码+测试通过）：
- `joints[] + constraints[]` 共存解析与去重。
- `revolute -> hinge` 收敛（JSON 资产已切到 `hinge`，`revolute` 保留兼容映射与弃用提示）。
- `angle_limit` 已切换到 Joint 路径并采用不等式 ALM 投影更新（`lambda <- max(0, lambda + mu*g)`）。
- 关节实现已统一到“原语组合”范式：
	- `Hinge = Point + AxisCross`
	- `Prismatic = PointOnAxis + Direction + RollLock`
	- `Cylindrical = PointOnAxis + Direction`
	- `Spherical = Point`
	- `Fixed = FixedPosition + FixedOrientation`
- `angular_velocity_drive` 工程参数落地：`max_torque`、`delay`、`delay_tau`，并实现 C1 平滑饱和。
- 线速度/位置/力驱动已落地并接入 `SceneLoader + IPCSolver` 运行时链路：
	- `linear_velocity_drive`（`target_velocity_mps/kv/max_force_n/delay_mps/delay_tau`）
	- `position_drive`（`target_position_m/kp/max_force_n/deadzone_m`）
	- `force_drive`（`body|body_b + force|axis*force_n + optional torque`）
- KPI 导出与门禁已落地：`V_w/T_sat/R_dual/max_constraint_violation`（CSV + 运行时阈值失败）。

未完成（仍属缺口）：
- `ConstraintSet / LegacyJointAdapter / CompositeJoint` 的统一装配层尚未单独显式成型（当前以原语化 Joint + SceneLoader 映射为主）。

结论：迁移规划核心主线（含驱动族 + DistanceLimit + DampedSpring）已完成；当前主要剩余为统一装配层工程化与参数规范化。

## 1. 当前已具备的核心能力（基线）

项目已经具备一个可运行的刚体隐式积分框架：
- 基本仿真循环（场景加载 → 求解步进 → CSV导出）
- 隐式时间积分（Implicit Euler / Implicit Newmark）
- 关节约束（Fixed / Hinge / Spherical / Prismatic / Cylindrical，且实现已原语化）
- 基础接触相关模块（Barrier、BroadPhase、CCD框架）
- 基础摩擦势函数与摩擦锥工具
- 单元测试框架（Catch2）和分模块测试
- 角速度驱动（`angular_velocity_drive`）及其饱和/延迟参数与 KPI 门禁

结论：当前更接近“研究原型 / 验证平台”，尚未达到“完整、工程级动力学仿真器”。

---

## 2. 关键缺失项（按完整仿真闭环拆解）

### A. 接触几何与距离查询尚未落地（高优先级）

**现状问题**
- `MeshShape::sdf()` 与 `gradient()` 仍是占位实现（以AABB中心近似）。
- 接触检测与法向力估计在 `IPCSolver` 中仍是简化距离近似（基于刚体质心距离）。

**影响**
- 难以得到可信的接触点、接触法向、穿透深度与接触雅可比。
- IPC势能与摩擦项无法在真实几何层面发挥作用。

**缺失内容**
1. 三角网格级别的最近点/最短距离查询（VV/VE/VF/EE）。
2. 稳定的接触法向与接触点恢复。
3. 与BVH联动的窄相检测与批量接触对生成。

---

### B. CCD 尚不完整（高优先级）

**现状问题**
- CCD中旋转只使用 `rot_t1 = rot_t0`（未积分角速度）。
- 候选对生成较粗，且与真实接触对筛选闭环不充分。

**影响**
- 对高速旋转体、细长体可能漏检碰撞。
- 时间步裁剪（TOI）可靠性受限。

**缺失内容**
1. 刚体旋转轨迹的连续碰撞检测（含角速度贡献）。
2. 更严格的TOI候选过滤与退化情形处理。
3. 与求解器步长控制的一致性策略（substep / rollback / safety margin自适应）。

---

### C. 摩擦模型与求解耦合不完整（高优先级）

**现状问题**
- `FrictionForm` 中切向位移采用简化估计，且 `dt=0.01` 硬编码。
- 法向力由近似 barrier gradient 估计，并非来自一致的接触约束/互补系统。
- 缺少粘-滑转换、历史切向位移追踪和稳定正则化策略。

**影响**
- 摩擦响应与时间步、速度尺度强相关，物理一致性不足。
- 难以稳定复现 stick-slip 行为。

**缺失内容**
1. 接触点历史（tangent basis + accumulated displacement）管理。
2. 与法向接触同源的摩擦约束/势能耦合。
3. 时间步一致化（消除硬编码dt，绑定仿真主dt）。

---

### D. 约束与关节的工程能力不足（中高优先级）

**现状问题**
- 关节实现虽然已统一为原语组合，但“约束集装配层”尚未独立显式化（ConstraintSet/Adapter/Composite）。
- 驱动接口已形成基础统一族（`angular_velocity_drive + linear_velocity_drive + position_drive + force_drive`），但参数标定与跨场景一致性规范仍待沉淀。
- 已具备 `DistanceLimit` 与 `DampedSpring` 的 JSON→SceneLoader→运行时闭环；`StrokeLimit` 等扩展限位仍待补齐。

**影响**
- 实际机构仿真中容易出现漂移、抖振，且难以调参。

**缺失内容**
1. 显式 `ConstraintSet/CompositeJoint` 组装层与分组统计接口。
2. `StrokeLimit` 等扩展限位原语实现与场景验证。
3. 驱动/软约束参数规范化与系统级回归基准（确保跨场景可调性）。

---

### E. 刚体动力学状态参数化仍偏原型（中优先级）

**现状问题**
- 优化变量采用 `[p, theta]` 增量方式，旋转更新与状态解释在模块间存在简化耦合。
- 转动惯量、世界系映射、角动量一致性校验链条不完整。

**影响**
- 长时仿真中可能出现能量漂移、姿态误差积累。

**缺失内容**
1. 更严格的旋转积分与角动量一致性处理。
2. 状态变量与物理量映射的统一接口（避免模块各自近似）。
3. 面向长期稳定性的数值守恒监测（能量、动量、约束残差）。

---

### F. 场景建模能力不足（中优先级）

**现状问题**
- Scene JSON 目前主要支持 body/joint/basic settings。
- 缺少网格资源加载、形状参数、材质参数、接触组、过滤规则等。

**影响**
- 无法支撑真实场景构建与批量复现实验。

**缺失内容**
1. 几何资源管线（mesh path / primitive shapes / scale / transform）。
2. 材质参数（摩擦、恢复系数、接触厚度、阻尼）与碰撞过滤。
3. 初始条件与控制输入脚本化。

---

### G. 求解器鲁棒性与性能工程不足（中优先级）

**现状问题**
- 线性求解器当前以 Eigen SimplicialLDLT 为主，未启用更强稀疏后端策略。
- Newton 收敛失败场景的回退策略较少（失败即返回）。
- 缺乏性能统计与可观测性（迭代数、残差、接触数量等）。

**影响**
- 大规模场景、强接触场景可扩展性和稳定性有限。

**缺失内容**
1. 预条件/多后端线性求解策略（含CHOLMOD路径验证）。
2. 失败恢复机制（trust region / regularization / damping schedule）。
3. 诊断指标导出与profiling。

---

### H. “完整仿真产品化”配套能力不足（中优先级）

**现状问题**
- 输出仅CSV，缺少标准轨迹/状态格式与可视化对接层。
- 测试以组件正确性为主，缺少系统级回归（场景基准曲线、能量守恒阈值、碰撞一致性）。

**影响**
- 难以保证版本演进中的物理行为一致性。

**缺失内容**
1. 端到端基准场景回归测试（单摆、双摆、堆叠、滑动摩擦、弹跳）。
2. 与参考解/外部引擎对比工具的自动化CI。
3. 结果可视化与指标门禁（误差阈值、性能阈值）。

---

## 3. 建议的落地优先级（最小可行“完整仿真”路线）

### P0（必须先做）
1. 网格级真实距离/法向/接触点查询（替换占位SDF）。
2. CCD引入角速度旋转轨迹并与步长控制闭环。
3. 摩擦与法向接触一致耦合，去除硬编码 `dt`。

### P1（稳定可用）
1. 求解器失败回退和诊断指标。
2. 场景格式扩展（几何与材质）。
3. 驱动参数规范与跨场景回归基准。

### P2（工程化）
1. 系统级回归基准与CI门禁。
2. 性能优化（线性求解后端、并行化、缓存策略）。
3. 标准化结果输出与可视化联动。

---

## 4. 一句话结论

NexDynIPC 已具备“隐式刚体+约束+接触框架”的骨架，但距离“完整动力学仿真系统”还缺少**真实几何接触闭环、完整CCD、物理一致摩擦耦合、工程级约束与求解鲁棒性**这四大核心能力。
