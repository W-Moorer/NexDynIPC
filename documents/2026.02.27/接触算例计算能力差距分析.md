## 二、差距分析：距离"完整接触算例"还差什么？

要让项目能够进行真正意义上**可信赖的接触算例计算**，以下是需要解决的核心差距：

---

### 差距 1：网格级真实几何接触检测（最核心缺失）

**当前状态**：
- `MeshShape::sdf()` 和 `gradient()` 是占位实现，仅使用 AABB 中心近似
- `IPCSolver::detectContacts()` 使用极其简化的刚体质心距离近似
- 无法获得精确的接触点、接触法向、穿透深度

**影响**：
- IPC 屏障势能无法在真实几何层面发挥作用
- 摩擦力的法向力计算不准确
- 接触响应与真实物理行为差距较大

**缺失内容**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 三角网格最近点查询 | P0 | VV (顶点-顶点), VE (顶点-边), VF (顶点-面), EE (边-边) |
| 接触法向计算 | P0 | 基于最近点几何的法向恢复 |
| 接触点位置恢复 | P0 | 碰撞对的实际接触点坐标 |
| 穿透深度计算 | P0 | 用于屏障激活和刚度计算 |
| BVH 窄相检测集成 | P0 | 与 BroadPhase 联动的精确检测 |

**参考实现**：
- [ipc-toolkit](https://github.com/ipc-sim/ipc-toolkit) 的 `collision` 模块
- Rigid-IPC 的几何查询实现

---

### 差距 2：完整的连续碰撞检测 (CCD)

**当前状态**：
- CCD 框架存在，但旋转积分不完整
- `IPCSolver.cpp` 第 167 行 `ccd_system_.computeEarliestTOI()` 使用简化模型
- 旋转只使用 `rot_t1 = rot_t0`，未考虑角速度贡献

**影响**：
- 对高速旋转体、细长体可能漏检碰撞
- 时间步裁剪（TOI）可靠性不足
- 可能导致穿透而非接触

**缺失内容**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 刚体旋转轨迹 CCD | P0 | 考虑角速度的连续旋转检测 |
| TOI 候选过滤优化 | P1 | 减少冗余检测，提高效率 |
| 退化情形处理 | P1 | 边-边平行、点-面接近等边界情况 |
| 与求解器步长控制闭环 | P1 | substep / rollback / safety margin 自适应 |

---

### 差距 3：物理一致的摩擦耦合

**当前状态**：
- `FrictionForm` 中切向位移使用简化估计
- `dt = 0.01` 硬编码在代码中
- 法向力由近似 barrier gradient 估计，非来自一致约束系统

**影响**：
- 摩擦响应与时间步、速度尺度强相关
- 难以稳定复现 stick-slip (粘滑) 行为
- 物理一致性不足

**缺失内容**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 接触点历史追踪 | P0 | tangent basis + accumulated displacement |
| 法向-摩擦同源耦合 | P0 | 与接触约束一致的摩擦势能 |
| 时间步参数统一化 | P0 | 去除硬编码 dt，绑定仿真主 dt |
| 粘滑转换模型 | P1 | 基于阈值的粘滞/滑动切换 |

---

### 差距 4：Barrier 势能与求解器集成

**当前状态**：
- `BarrierPotential` 有基础实现 (value/gradient/hessian)
- `AutoDiffBarrierForm` 支持自定义 barrier 函数
- 但 barrier 未被真正集成到求解器优化流程中

**影响**：
- 接触势能未真正参与 Newton 迭代
- 无法产生实际的接触排斥力

**缺失内容**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| ContactForm / BarrierForm | P0 | 作为 Form 集成到 SimulationProblem |
| 接触势能梯度/Hessian | P0 | 正确的一阶/二阶导数 |
| 接触雅可比计算 | P0 | 用于约束/梯度计算 |

---

### 差距 5：场景建模与参数配置

**当前状态**：
- JSON 场景文件主要支持 body / joint / basic settings
- 缺少几何资源路径、形状参数、材质参数

**缺失内容**：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 几何资源管线 | P1 | mesh path / primitive shapes / scale / transform |
| 材质参数 | P1 | 摩擦系数 μ、恢复系数 e、接触阻尼 |
| 碰撞过滤 | P1 | collision group / mask |
| 初始条件脚本化 | P2 | 初始位置、速度、控制系统 |

### 距离"完整接触算例计算"的核心差距总结

1. **网格级真实几何接触检测** - 最核心缺失，无法获得精确接触信息
2. **Barrier 势能未集成** - 接触排斥力未真正参与求解
3. **CCD 旋转积分不完整** - 高速运动可靠性不足
4. **摩擦耦合不一致** - 法向力/切向力未同源计算

### 建议路线

1. **短期（1-2周）**: 快速实现网格级最近点查询 + 简化 barrier 集成
2. **中期（3-4周）**: 完整 CCD + 摩擦耦合 + 场景扩展
3. **长期（2-3月）**: 鲁棒性优化 + 工程化完善

## 五、结论

NexDynIPC 项目已经具备了一个优秀的刚体动力学仿真框架骨架，但要实现**真正意义上的接触算例计算**（能够产生可信的接触力、避免穿透、处理摩擦），还需要在以下四个核心方向进行重点投入：

1. **真实几何接触检测** - 替换占位的质心距离近似
2. **Barrier 势能集成** - 让接触参与 Newton 迭代
3. **完整 CCD** - 考虑旋转的连续碰撞检测
4. **物理一致摩擦** - 法向力与摩擦力同源耦合

完成这些核心功能后，项目将能够支持基本的刚体接触仿真场景（如刚体堆叠、碰撞、滑动等）。
